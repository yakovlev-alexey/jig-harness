# Decisions — Spec-Driven Workflow

## 2026-07-06 — Native jig skills over external spec CLIs

**Decision:** Adopt the native spec-driven workflow spine (jig skills + flat living
specs + coarse `spec-present` gate) documented in
[ADR 0001 — Spec-driven workflow](../../adr/0001-spec-driven-workflow.md).

**Alternatives considered:** See ADR 0001 — external spec CLIs (OpenSpec, Spec Kit),
guidance-only skills without a gate, and slice-scoped specs with enforced mapping.

**Rationale:** See ADR 0001. This feature adopts that project-wide decision; the
alternatives and tradeoffs are recorded there to avoid duplication.
