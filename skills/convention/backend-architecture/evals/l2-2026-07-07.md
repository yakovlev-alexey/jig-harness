## Case — create-user with duplicate-email guard

**Prompt:** Add a `create-user` usecase that checks for duplicate emails before insert.

**Oracles (objective):**

| Check                                                  | Mechanism                                             | Rule ID                         |
| ------------------------------------------------------ | ----------------------------------------------------- | ------------------------------- |
| Command and query composed in usecase, no cross-import | `@jig-harness/no-command-query-cross-calls` ESLint    | be-no-command-query-cross-calls |
| Domain has no I/O                                      | `@jig-harness/domain-no-io` ESLint                    | be-domain-no-io                 |
| Endpoint → usecase → command/query flow                | `evals/oracle-check.mjs` import-path structural check | be-layer-flow                   |
| Kebab-case layer filenames                             | `unicorn/filename-case` ESLint + structural check     | be-kebab-case                   |
