# L2 application eval — react-composition

Date: 2026-07-07  
Case: widget reuses presentational UI owned by another widget  
Oracle: `evals/l2-oracle.sh` (structural checks + `pnpm exec eslint` with template frontend config)

## Prompt

> Add a widget that reuses a presentational block owned by another widget.

## Oracle (objective)

`evals/l2-oracle.sh <fixture>` copies the fixture into a temp `src/` tree and checks:

| Check                                                                     | Rule ID                                    |
| ------------------------------------------------------------------------- | ------------------------------------------ |
| No `*.widget.tsx` imports another widget entry                            | **rc-no-widget-imports-widget**            |
| Each page/component/widget-ui TSX imports colocated `./<name>.css`        | **rc-colocated-css**                       |
| Each widget folder has a `*.widget.tsx` entry                             | **rc-widget-suffix**                       |
| ESLint `boundaries/element-types` on temp tree (same as template dogfood) | **rc-no-widget-imports-widget** (enforced) |

Fixtures: `evals/fixtures/widget-reuse-with-skill/` (compliant) and
`evals/fixtures/widget-reuse-without-skill/` (widget cross-import anti-pattern).

## Results

| Baseline          | Fixture                      | Oracle   | Notes                                                                                                         |
| ----------------- | ---------------------------- | -------- | ------------------------------------------------------------------------------------------------------------- |
| **without_skill** | `widget-reuse-without-skill` | **FAIL** | `signup-cta.widget.tsx` imports `feature-highlight.widget.tsx` (`rc-no-widget-imports-widget`)                |
| **with_skill**    | `widget-reuse-with-skill`    | **PASS** | Shared `PromoBadge` lifted to `components/`; page composes both widgets; colocated CSS + `.widget.tsx` suffix |

### Delta

With the skill, the agent lifts shared presentational markup into `components/` (or composes widgets on the page) instead of importing one widget entry from another. The objective oracle moves from **FAIL → PASS** on `rc-no-widget-imports-widget` and structural composition checks.

## Reproduce

```bash
pnpm install
./skills/convention/react-composition/evals/l2-oracle.sh \
  skills/convention/react-composition/evals/fixtures/widget-reuse-with-skill
./skills/convention/react-composition/evals/l2-oracle.sh \
  skills/convention/react-composition/evals/fixtures/widget-reuse-without-skill
```

Expected: first exits 0, second exits 1 citing `rc-no-widget-imports-widget`.

## without_skill method

Baseline output is the `widget-reuse-without-skill` fixture: a plausible default where the new widget internally imports the sibling widget entry to reuse its UI. Recorded by authoring the anti-pattern fixture and grading it with the same oracle (no live subagent run).
