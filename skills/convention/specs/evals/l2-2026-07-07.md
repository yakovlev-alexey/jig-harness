# L2 application eval — specs

Date: 2026-07-07  
Oracle: `node evals/oracle-check-spec-structure.mjs <spec-path>` (structural/file check)  
Prompt: _Write `docs/specs/user-registration/spec.md` for a duplicate-email rule._

## Oracle checklist

| Check                                                                     | Rule                     |
| ------------------------------------------------------------------------- | ------------------------ |
| Feature-scoped folder name (intent, not slice)                            | `wf-spec-feature-scoped` |
| `## Purpose` present                                                      | `spec.md` structure      |
| Numbered `### R<n>` requirements with `SHALL`/`MUST`                      | `wf-spec-gwt`            |
| Every requirement has ≥1 `#### Scenario` with **GIVEN**/**WHEN**/**THEN** | `wf-spec-gwt`            |
| No decision rationale duplicated in spec                                  | `wf-decisions-colocated` |

Script exits `0` only when all checks pass; any failure prints `FAIL: …` to stderr.
