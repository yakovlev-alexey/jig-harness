## Case 1 — Shared UI filter for two widgets on one page

**Prompt:** A shared UI filter is needed by two widgets on one page — wire it.

### without_skill — unverified — design-time

Typical output without the skill: lift filter state to the page and pass props, or import
`useStore(usersFilterAtom)` inside each widget-ui presenter file.

```typescript
// src/slices/users/widgets/user-table/user-table.tsx — BAD (presenter owns store hook)
import { useStore } from '@nanostores/react';
import { usersFilterAtom } from '../../store/model/users-filter-store';

export function UserTableUi() {
  const filter = useStore(usersFilterAtom);
  // ...
}
```

**Oracle:** `pnpm --filter @jig-harness/eslint-config exec vitest run sd-no-store-in-presentational.test.js`
equivalent lint on the snippet above → **2** `no-restricted-imports` violations
(`@nanostores/react` + store model import; messages include `sd-no-store-in-presentational`).

Also violates **sd-no-server-data-in-stores** if the agent mirrors the API list into the
filter atom (guidance-only; not lint-enforced).

### with_skill

Filter lives in a Nano Store atom; hooks live in `*.widget.tsx` containers; presenters
take props only.

```typescript
// src/slices/users/store/model/users-filter-store.ts
import { atom } from 'nanostores';
export const usersFilterAtom = atom('');

// src/slices/users/widgets/user-table/user-table.widget.tsx — container
import { useStore } from '@nanostores/react';
import { useQuery } from '@tanstack/react-query';
import { usersFilterAtom } from '../../store/model/users-filter-store';
import { usersQuery } from '../../store/queries/users-query';
import { filterUsers } from '../../store/selectors/filter-users';
import { UserTableUi } from './user-table';

export function UserTableWidget() {
  const filter = useStore(usersFilterAtom);
  const { data } = useQuery(usersQuery());
  const users = data ? filterUsers(data, filter) : [];
  return <UserTableUi users={users} />;
}

// src/slices/users/widgets/user-table/user-table.tsx — presenter, props only
type UserTableUiProps = { users: UserResponse[] };
export function UserTableUi({ users }: UserTableUiProps) {
  // render from props — no store or data imports
}
```

**Oracle:** lint on container + presenter snippets → **0** `no-restricted-imports` violations.

### Delta

Skill routes shared UI state to Nano Stores and keeps store/data hooks in widget entries;
lint objectively rejects the without_skill presenter pattern and accepts the with_skill split.

## Case 2 — Create a user and refresh the list

**Prompt:** Create a user and refresh the list.

### without_skill — unverified — design-time

Typical output: call the command `fetch` directly inside a click handler, or use
`useMutation` without invalidating the list query key.

```typescript
// src/slices/users/widgets/create-user/create-user.widget.tsx — BAD
import { useMutation } from '@tanstack/react-query';
import { createUserCommand } from '../../store/commands/create-user-command';

export function CreateUserWidget() {
  const mutation = useMutation({ mutationFn: createUserCommand });
  // missing onSuccess → list stays stale after create
}
```

**Oracle (structural):** file lacks `onSuccess` callback that references exported
`usersQueryKey` (or `queryClient.invalidateQueries({ queryKey: usersQueryKey })`) → **fail**.

### with_skill

```typescript
// src/slices/users/store/queries/users-query.ts
export const usersQueryKey = ['users'] as const;
export function usersQuery() {
  return queryOptions({ queryKey: usersQueryKey, queryFn: fetchUsers });
}

// src/slices/users/widgets/create-user/create-user.widget.tsx — GOOD
import { useMutation, useQueryClient } from '@tanstack/react-query';
import { createUserCommand } from '../../store/commands/create-user-command';
import { usersQueryKey } from '../../store/queries/users-query';

export function CreateUserWidget() {
  const queryClient = useQueryClient();
  const mutation = useMutation({
    mutationFn: createUserCommand,
    onSuccess: () => queryClient.invalidateQueries({ queryKey: usersQueryKey }),
  });
  // ...
}
```

**Oracle (structural):**

- `useMutation({ mutationFn:` present → **pass**
- `onSuccess` invalidates exported `usersQueryKey` → **pass**

### Delta

Skill enforces **sd-mutations-usemutation** with cache invalidation on success; the
without_skill pattern leaves stale server state; the with_skill pattern satisfies the
structural oracle.
