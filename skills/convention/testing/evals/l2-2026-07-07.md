## Case 1 — duplicate-email edge-case placement

### Oracle checklist (objective)

If you want faster feedback, a React Testing Library render of the signup form with a mocked `fetch` is fine for the duplicate case.

Reserve **E2E** for main happy paths only (`test-e2e-happy-paths`); edge cases belong in unit/integration. No RTL/jsdom — frontend integration uses Playwright + MSW.

## Case 2 — Postgres down in CI

**Query:** "Postgres is down in CI — can I skip the integration tests?"

### Oracle checklist (objective)

| #   | Check                                                                                  | Rule / source                             |
| --- | -------------------------------------------------------------------------------------- | ----------------------------------------- |
| 1   | Answer is **no** — do not skip integration tests                                       | `test-db-required`                        |
| 2   | Does **not** recommend `test.skip`, conditional skip, or ignoring DB failures          | `test-db-required` (Red Flags)            |
| 3   | Names fixing **`DATABASE_URL`** and/or running **`pnpm db:setup`** (migrated Postgres) | `test-db-required`, Verify vs E2E section |

### without_skill

> Yes — wrap integration tests in `test.skip(!process.env.DATABASE_URL)` or mark the job `continue-on-error` when Postgres is unavailable. CI can run unit tests only and re-enable integration once the database service is back.

| #   | Pass                             |
| --- | -------------------------------- |
| 1   | no — explicitly permits skipping |
| 2   | no — recommends `test.skip`      |
| 3   | no — no fix path for DB setup    |

**Score: 0/3**

### with_skill

> **No.** A working database is mandatory — missing or broken Postgres is a **hard failure**, never skip or `test.skip` (`test-db-required`). Fix `DATABASE_URL`, run `pnpm db:setup` so the schema is migrated, and ensure CI provisions Postgres before `pnpm verify`. Integration and E2E both require real Postgres; skipping when DB is down violates the letter and spirit of the rules.

| #   | Pass |
| --- | ---- |
| 1   | yes  |
| 2   | yes  |
| 3   | yes  |

**Score: 3/3**

### Delta

`with_skill` enforces `test-db-required` (hard failure + `db:setup`/`DATABASE_URL`). `without_skill` recommends `test.skip` and `continue-on-error` — exactly the anti-pattern listed under Red Flags.
